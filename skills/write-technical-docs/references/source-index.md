# Google style source index

Use this index to verify exact, uncommon, or potentially changing guidance. Browse the live Google page instead of relying on a remembered rule when the distinction affects the deliverable.

## Authority and lookup policy

Apply this precedence:

1. Explicit user requirements and project-specific style
2. Google Developer Documentation Style Guide
3. An appropriate third-party reference when the first two sources are silent

Treat the guide as guidance. Depart when clarity, reader needs, technical correctness, or consistent project usage justifies an exception.

These files paraphrase and attribute the [Google Developer Documentation Style Guide](https://developers.google.com/style), whose page content is published under the [Creative Commons Attribution 4.0 License](https://creativecommons.org/licenses/by/4.0/) unless otherwise noted. Follow each source page's license notice for exceptions and code samples.

Do not copy the complete Google word list or product-name catalog into this skill. Those resources are broad and change over time. Query the live pages for:

- disputed spelling, hyphenation, word choice, or preferred terminology;
- current Google product names and capitalization;
- Android-only or Google Cloud-only annotations;
- a rule whose exact exceptions matter;
- guidance that conflicts with current project usage.

Record a material exception in the handoff when reviewers need to preserve it.

## Source maintenance

When refreshing this skill:

1. Review **What's new** and the update dates on the relevant official pages.
2. Compare the changed guidance with the matching task reference.
3. Update only stable, reusable summaries; keep volatile lists as live lookups.
4. Preserve the separation between workflow in `SKILL.md` and details in `references/`.
5. Check every link and remove or replace redirects and retired pages.
